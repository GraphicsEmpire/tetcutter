\hyperlink{structThread}{Thread} local storage keyword. \hyperlink{structA}{A} variable that is declared with the {\ttfamily \+\_\+\+Thread\+\_\+local} keyword makes the value of the variable local to each thread (known as thread-\/local storage, or T\+L\+S). \hyperlink{classExample}{Example} usage\+: 
\begin{DoxyCode}
1 // This variable is local to each thread.
2 \_Thread\_local int variable;
\end{DoxyCode}
 \begin{DoxyNote}{Note}
The {\ttfamily \+\_\+\+Thread\+\_\+local} keyword is a macro that maps to the corresponding compiler directive (e.\+g. {\ttfamily \+\_\+\+\_\+declspec(thread)}). 

This directive is currently not supported on Mac O\+S \hyperlink{structX}{X} (it will give a compiler error), since compile-\/time T\+L\+S is not supported in the Mac O\+S \hyperlink{structX}{X} executable format. Also, some older versions of Min\+G\+W (before G\+C\+C 4.\+x) do not support this directive. 
\end{DoxyNote}
