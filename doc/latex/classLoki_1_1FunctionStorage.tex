\subsection{Detailed Description}
\subsubsection*{template$<$class T$>$class Loki\+::\+Function\+Storage$<$ T $>$}

Implementation of the Storage\+Policy used by \hyperlink{classLoki_1_1SmartPtr}{Smart\+Ptr}. 

This storage policy is used by \hyperlink{classLoki_1_1SmartPointer}{Smart\+Pointer} \hyperlink{classLoki_1_1CachedFactory}{Cached\+Factory}\textquotesingle{}s encapsulation policy. It\textquotesingle{}s purpose is to call a \hyperlink{classLoki_1_1Functor}{Functor} instead of deleting the underlying pointee object. You have to set the callback functor by calling \hyperlink{classLoki_1_1FunctionStorage_ac55c8a149c74a27751204b0215c76041}{Set\+Call\+Back\+Function(const Functor\+Type \&functor)}.

Unfortunately, the functor argument is not a reference to the \hyperlink{classLoki_1_1SmartPtr}{Smart\+Ptr} but a void $\ast$. Making functor argument a reference to the pointer would require the \hyperlink{classLoki_1_1FunctionStorage}{Function\+Storage} template to know the full definition of the \hyperlink{classLoki_1_1SmartPtr}{Smart\+Ptr}.
