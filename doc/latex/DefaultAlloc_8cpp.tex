\subsection{Detailed Description}
Tests compiler compliance when using the new and delete operators.

\begin{DoxyAuthor}{Author}
Rich Sposato (original author)
\end{DoxyAuthor}
\begin{DoxyParagraph}{Purpose of Program}
This program tests how well C++ compilers comply with the I\+S\+O C++ Standard in implementing new, new \mbox{[}\mbox{]}, delete, and delete \mbox{[}\mbox{]} operators. The program should identify each known way in which a C++ compiler can fail to comply. It provides versions of each new, new\mbox{[}\mbox{]}, delete, and delete \mbox{[}\mbox{]} operators that classes typically overload. These functions have the same signatures as those in the \hyperlink{namespaceLoki}{Loki} project, and the same as those enumerated in section 18 of the I\+S\+O C++ Standard.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Running the Program}
If the programs fails to compile, or crashes when executed, the compiler does not comply with the I\+S\+O C++ Standard. As the program runs, it should\+:
\begin{DoxyItemize}
\item identify your compiler,
\item execute each version of the new, new\mbox{[}\mbox{]}, delete, and delete \mbox{[}\mbox{]} operators,
\item determine if compiler provides correct behavior when exceptions are thrown,
\item determine if correct operators were called for each appropriate situation,
\item and determine if the correct parameters were provided to each function.
\end{DoxyItemize}
\end{DoxyParagraph}

\begin{DoxyParagraph}{Connection to Loki}
This file is provided as part of the \hyperlink{namespaceLoki}{Loki} project -\/ but does not depend upon any other files in \hyperlink{namespaceLoki}{Loki}. It was made for \hyperlink{namespaceLoki}{Loki} by Rich Sposato to determine which compilers can safely use \hyperlink{namespaceLoki}{Loki}\textquotesingle{}s Small-\/\+Object Allocator. During his work on \hyperlink{namespaceLoki}{Loki}, Rich noticed that some compilers did not provide the correct values to the allocation / deallocation operators, or worse, called the wrong operator. Rather than make several variations of allocator functions to work around areas of non-\/compliance, Rich decided to write a program that exposes which compilers do not comply with the I\+S\+O C++ Standard. It is recommended that you execute this program before incorporating \hyperlink{namespaceLoki}{Loki}\textquotesingle{}s Small-\/\+Object Allocator from your source code. Hence, this program is a \char`\"{}crash test dummy\char`\"{} for how well your compiler can handle \hyperlink{namespaceLoki}{Loki}. 
\end{DoxyParagraph}
