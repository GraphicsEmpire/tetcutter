Called by client to indicate a change in the number of non-\/\+R\+M\+L threads that are running. 

This is a performance hint to the R\+M\+L to adjust how many threads it should let run concurrently. The delta is the change in the number of non-\/\+R\+M\+L threads that are running. For example, a value of 1 means the client has started running another thread, and a value of -\/1 indicates that the client has blocked or terminated one of its threads.

Request that connection to server be closed. 

Causes each job associated with the client to have its cleanup method called, possibly by a thread different than the thread that created the job. This method can return before all cleanup methods return. Actions that have to wait after all cleanup methods return should be part of client\+::acknowledge\+\_\+close\+\_\+connection. Pass true as exiting if \hyperlink{classrml_1_1server_a3119d8d6c2d17932610653f6fda4f87a}{request\+\_\+close\+\_\+connection()} is called because exit() is called. In that case, it is the client\textquotesingle{}s responsibility to make sure all threads are terminated. In all other cases, pass false.
